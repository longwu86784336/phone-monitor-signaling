# PhoneMonitor 使用说明 v3

## 文件说明

| 文件 | 用途 |
|------|------|
| `collector-v3.apk` (7.1MB) | 安装在采集端手机（旧手机） |
| `viewer-v3.apk` (7.1MB) | 安装在查看端手机（另一台手机） |
| `server.js` | 信令服务器（运行在你的电脑上） |

---

## 第一步：启动信令服务器

**在你的电脑上运行：**

```powershell
cd "D:\软件\安装包\手机\PhoneMonitor\signaling-server"
node server.js
```

成功后显示：
```
Phone Monitor 信令服务器 v3
端口: 3000
采集端注册: POST /api/register
采集端上传: POST /api/upload
查看端拉取: GET  /api/latest-image?deviceId=xxx
```

> ⚠️ 保持此窗口运行，不要关闭！

---

## 第二步：确认电脑IP地址

打开 PowerShell 运行：
```powershell
ipconfig
```

找到你的IP地址（例如 `192.168.199.105`）

---

## 第三步：安装采集端（旧手机）

1. 把 `collector-v3.apk` 传到旧手机，安装
2. 打开 App
3. 修改**服务器地址**为你的电脑IP（格式：`192.168.199.105:3000`）
4. 首次使用：设置密码（6-16位），点"设置密码并启动"
5. 再次打开：输入密码，点"启动监控"
6. 屏幕上显示**二维码** ✅

---

## 第四步：安装查看端（另一台手机）

1. 把 `viewer-v3.apk` 传到另一台手机，安装
2. 打开 App
3. 用摄像头**扫描采集端显示的二维码**
4. 扫码成功后输入密码，点"连接监控"
5. 画面开始显示 ✅

---

## 异地监控（两台手机在不同网络）

### 方案A：用热点（最简单）
1. 用一台手机开热点
2. 另一台手机和你的电脑都连这个热点
3. 用热点分配给电脑的IP地址配置采集端

### 方案B：路由器端口映射（需要公网IP）
1. 登录路由器管理页面（通常是 `192.168.1.1`）
2. 找到"端口映射"或"虚拟服务器"
3. 添加规则：外部端口 `3000` → 内部IP `192.168.199.105` 端口 `3000`
4. 采集端服务器地址填：`你的公网IP:3000`

### 方案C：内网穿透（推荐，等服务器部署好后用）
我将把 `server.js` 部署到 **免费云服务器**，两台手机在任何网络下都能用。

## 排错

| 问题 | 解决方法 |
|------|---------|
| 二维码空白 | 清掉App数据重开，或检查密码是否已设置 |
| 扫码没反应 | 确保两台手机在同一WiFi，服务器正在运行 |
| 画面不显示 | 检查采集端是否"启动监控"成功，查看logcat |
| 连接超时 | 检查防火墙是否允许3000端口，关闭电脑防火墙试试 |

---

## 息屏保活

采集端启动后会显示前台通知"手机监控正在运行"，息屏后继续工作。
如需开机自启：重启手机后会自动启动服务（需要授予自启动权限）。
